tools.py

Single source of truth — JSON schema definitions for all 151+ tools (name, description, input parameters, annotations). The AI reads these at startup to know what it can do.

server.py

MCP protocol entry point. Handles tools/list and tools/call requests. Runs executor in a thread pool. Redacts passwords from logs.

tool_executor.py

Central dispatcher. Maps every tool name to the right api_client method. Builds complex nested payloads (e.g. create_profile) from flat AI inputs.

api_client.py

HTTP layer. Manages session state. Uses different auth headers per API version: IDS-SESSION-ID, icSessionId, INFA-SESSION-ID. Routes to regional pods.

Why It's Client-Agnostic
The server speaks the open MCP standard — it has zero knowledge of which AI is calling it.
// The MCP protocol has just two operations:

// 1. AI asks: "what can you do?"
client → "tools/list"              // request
server → [{ name, description, inputSchema }, ...]  // 151+ tools

// 2. AI asks: "do this specific thing"
client → "tools/call" { name: "idmc_list_connections", arguments: {} }
server → { "content": [{ "type": "text", "text": "..." }] }
Setup & Registration
One-time setup — then register in your AI assistant's config file.
1

Install dependencies

From the project directory, install the required Python packages.

pip install -r requirements.txt
2

Configure credentials

Run the setup script — it opens a native OS dialog for your IDMC username and password. Credentials are saved to .mcp.json.

python setup_credentials.py
3

Register with your AI assistant

Add the server config to your AI tool's config file. The path and filename differ per client — see tabs below.

Usage Examples
Type natural language — the AI picks the right tools and assembles the API calls automatically.
🔍 Data Profiling Profiling
Profile the CUSTOMERS table on my Oracle connection and show me null percentages
// Tools called in sequence:
1. idmc_login         // authenticate
2. idmc_list_connections // find Oracle connection
3. idmc_get_object_fields // discover columns
4. idmc_create_profile   // create profile task
5. idmc_run_profile      // execute it
6. idmc_list_columns     // get null% stats
👤 User Management Admin
Create a new user john@acme.com and add them to the Data Engineers group
// Tools called in sequence:
1. idmc_login          // authenticate
2. idmc_get_user_groups // find group ID
3. idmc_create_user    // create the user
4. idmc_add_users_to_group // assign group
⚡ Run a DI Task Data Integration
Start the mapping task "Load_Salesforce_to_Snowflake" and monitor it
// Tools called in sequence:
1. idmc_login         // authenticate
2. idmc_list_assets   // find task by name
3. idmc_start_di_job  // start job (type: MTT)
4. idmc_get_activity_log // monitor status
📅 Create a Schedule Admin
Create a daily schedule at 6 AM New York time starting tomorrow
// Tools called:
1. idmc_login           // authenticate
2. idmc_create_schedule  // schedule payload:

{
  "name": "Daily 6AM NY",
  "interval": "Daily",
  "start_time": "2026-06-20T10:00:00Z",
  "timezone": "America/New_York"
}
📦 Export Assets Migration
Export all mappings in project "Finance" to a ZIP file
// Tools called in sequence:
1. idmc_login           // authenticate
2. idmc_list_project_assets // find all assets
3. idmc_start_export_job    // kick off export
4. idmc_get_export_job_status // poll status
5. idmc_download_export_package // get ZIP
🔒 Permissions Security
Give the "Analysts" group read and execute access to project "Sales_DQ"

Extending the Server

Adding a new IDMC capability is always three files:

# Step 1 — tools.py: add the tool schema
{
  "name": "idmc_my_new_tool",
  "description": "What it does",
  "input_schema": { "type": "object", "properties": { ... } }
}

# Step 2 — tool_executor.py: map the name → API call
elif name == "idmc_my_new_tool":
    return self.client.my_new_method(inputs.get("param"))

# Step 3 — api_client.py: implement the HTTP call
def my_new_method(self, param):
    url = f"{self.session.base_url}/api/v2/my-endpoint"
    return self._get(url, self._di_headers())
